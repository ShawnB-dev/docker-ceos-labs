# ラボ 09 – セキュリティ (AAA、ACL、CoPP、および管理プレーンの要塞化)

このラボでは、cEOS 管理ネットワーク全体に基本的なセキュリティ制御を実装しました。  
私の目標は、最新のデータセンター環境で管理プレーンを保護し、アクセスを制限し、AAA を強制し、コントロールプレーンの保護を適用する方法を実証することでした。

このラボは、以前の自動化とテレメトリの作業 (ラボ 07–08) に基づいて構築されており、同じファブリックに実世界のセキュリティのベストプラクティスを適用する方法を示しています。

---

## 1. 概要

このラボでは、以下を実装しました：

- AAA (ローカル認証 + ロールの分離)
- 管理プレーン ACL (mgmt-acl)
- インフラストラクチャ ACL (iACL)
- SSH の強化 (暗号、キー交換、アイドルタイムアウト)
- eAPI アクセスの制限
- コントロールプレーン ポリシング (CoPP)
- シスログ転送
- ログインバナーとコンプライアンス テキスト

これらの制御は、あらゆる本番ネットワークのベースライン セキュリティ ポスチャを形成します。

---

## 2. トポロジー

このラボでは、ラボ 08 と同じ管理専用トポロジーを使用します：

mgmt-net (10.99.0.0/24)
┌──────────┬──────────┬──────────┬──────────┐
│          │          │          │          │
10.99.0.11 10.99.0.12 10.99.0.13 10.99.0.14  10.99.0.50
ceos-1     ceos-2     ceos-3     ceos-4    rust-host
│          │          │          │          │
└──────────┴──────────┴──────────┴──────────┴──────────┘

- **ceos-1 → ceos-4**: セキュリティ ベースラインが適用されるデバイス
- **rust-host (10.99.0.50)**: 唯一許可された管理クライアント (自動化 + テレメトリ)

---

## 3. 実装されたセキュリティ制御

### AAA + RBAC
私は 2 つのロールを作成しました：
- **admin** (権限レベル 15)
- **ops** (権限レベル 5)

AAA はローカルの認証と認可のために構成されています。

### 管理 ACL (mgmt-acl)
自動化/テレメトリ ホスト (10.99.0.50) のみが以下にアクセスすることを許可されています：
- SSH
- eAPI (HTTPS/JSON-RPC)

他のすべてのソースは拒否され、ログに記録されます。

### インフラストラクチャ ACL (iACL)
管理インターフェースを不要なトラフィックから保護します：
- ICMP を許可
- 確立された TCP セッションを許可
- 管理サブネットから送信されたトラフィックを拒否 (アンチスプーフィング)
- その他すべてを許可

### SSH の強化
以下を強制しました：
- 強力な暗号
- 強力なキー交換
- アイドル タイムアウト
- パスワード認証のみ (ラボ環境用)

### eAPI の強化
eAPI は mgmt-acl によって制限され、デフォルト VRF でのみ有効になっています。

### CoPP (コントロール プレーン ポリシング)
単純な 3 クラスの CoPP ポリシーを適用しました：
- Critical: 2000 pps
- Important: 1000 pps
- Normal: 500 pps

### ロギング
すべてのデバイスはログを 10.99.0.50 に転送します。

## 4. ラボの実行

環境を開始します：
```bash
docker compose up -d
```

## 5. 検証チェックリスト
私は以下のチェックを使用してセキュリティ ポスチャを検証しました：

検証コマンド例：
`show aaa authentication`
`show ip access-lists mgmt-acl`
`show policy-map copp-system`
