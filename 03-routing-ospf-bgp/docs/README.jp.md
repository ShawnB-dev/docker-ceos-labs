# ラボ 03 – OSPF アンダーレイ、iBGP オーバーレイ、および eBGP エッジ

## 1. 概要

このラボでは、3 台の cEOS ルータと 1 台のエッジ ルータを使用して、現実的なルーティング ファブリックを構築しました。

- 私はすべてのポイントツーポイント リンク間で OSPF をアンダーレイ IGP として構成しました。
- 私はオーバーレイ制御プレーンのためにループバック間で iBGP フルメッシュを確立しました。
- 私は小さな「ISP」エッジ ルータへの eBGP ピアリングを設定し、それがパブリック スタイルのプレフィックスをアドバタイズします。

このパターンは、最新のエンタープライズおよびデータセンター設計 (アンダーレイ/オーバーレイ) を反映しており、内部ルーティングが外部 BGP エッジとどのように相互作用するかを示しています。

## 2. トポロジ

- 私は `ceos-1`、`ceos-2`、`ceos-3` をルーテッド トライアングルを形成するように構成しました。
- 私はすべてのスイッチ間リンクで OSPF エリア 0 を有効にしました。
- 私はルータ ID および iBGP アップデート ソースとしてループバックを使用しました。
- 私は `edge` ルータを eBGP 経由で `ceos-1` に接続し、それが `203.0.113.0/24` をアドバタイズします。
```text
                       +----------------------+
                       |       ceos-1         |
                       |   Loopback 1.1.1.1   |
                       |   OSPF Area 0        |
                       +----+------------+----+
                            |            |
                            |            |
                     10.0.12.1/30   10.0.31.1/30
                            |            |
                            |            |
                       +----+------------+----+
                       |       ceos-2         |
                       |   Loopback 2.2.2.2   |
                       +----+------------+----+
                            |            |
                            |            |
                     10.0.23.1/30   10.0.23.2/30
                            |            |
                            |            |
                       +----+------------+----+
                       |       ceos-3         |
                       |   Loopback 3.3.3.3   |
                       +----------------------+
```
## 3. アドレス設計

- ループバック:
  - ceos-1: 1.1.1.1/32
  - ceos-2: 2.2.2.2/32
  - ceos-3: 3.3.3.3/32
  - edge: 203.0.113.1/32

- P2P リンク:
  - ceos-1 ↔ ceos-2: 10.0.12.0/30
  - ceos-2 ↔ ceos-3: 10.0.23.0/30
  - ceos-3 ↔ ceos-1: 10.0.31.0/30
  - ceos-1 ↔ edge: 10.0.14.0/30

- 管理:
  - ceos-1: 10.99.0.11
  - ceos-2: 10.99.0.12
  - ceos-3: 10.99.0.13
  - edge:   10.99.0.14

## 4. プロトコル設計

### OSPF アンダーレイ

- 私はすべての cEOS ルータでプロセス 1 を構成しました。
- 私はすべての P2P リンクとループバックをエリア 0 に配置しました。
- 私は検証のために以下のコマンドを使用しました:
  - `show ip ospf neighbor`
  - `show ip route ospf`
  - `show ip ospf database`

### iBGP オーバーレイ

- 私は `ceos-1`/`2`/`3` で AS 65000 を使用しました。
- 私はループバック間で iBGP フルメッシュを設定しました:
  - 1.1.1.1 ↔ 2.2.2.2
  - 1.1.1.1 ↔ 3.3.3.3
  - 2.2.2.2 ↔ 3.3.3.3
- 私は検証のために以下のコマンドを使用しました:
  - `show ip bgp summary`
  - `show ip bgp`
  - `show ip bgp neighbors`

### eBGP エッジ

- 私は `edge` ルータを AS 65001 に配置しました。
- 私は `ceos-1` (AS 65000) と `edge` (AS 65001) の間で eBGP を確立しました。
- `edge` ルータは `203.0.113.0/24` をアドバタイズします。
- 私は検証のために以下のコマンドを使用しました:
  - `show ip bgp` (ceos-1/2/3 上)
  - `show ip route 203.0.113.0/24`

## 5. クイック スタート

私は以下のコマンドを使用してラボを開始しました:
```bash
cd 03-routing
docker compose up -d

docker exec -it ceos-1 Cli
docker exec -it ceos-2 Cli
docker exec -it ceos-3 Cli
docker exec -it edge Cli
```
